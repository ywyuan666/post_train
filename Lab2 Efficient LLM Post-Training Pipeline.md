## **4. Assignment Tasks**

### **Part 1: LoRA Implementation (REQUIRED)**
*All students must complete this part.*

1.  **Task:** Implement the LoRA adapter layers manually within a Transformer block (specifically on Query and Value projection matrices).
    *   **Vibe Coding:** You are encouraged to use AI assistants to generate the boilerplate structure, but you **must verify** the mathematical implementation (scaling factors, weight freezing).
    *   Do **not** use `peft.LoraConfig` for the core adapter logic. You must define the $A$ and $B$ matrices and the forward pass logic yourself.
2.  **Training:** Fine-tune the base model on the **Alpaca/TinyShakespeare** subset using your custom LoRA implementation.
    *   **Constraint:** Training must complete within **1 hour**.
3.  **Validation:** Generate responses for 5 held-out instructions. Compare them against the base model (pre-tuning). Verify that training loss decreased.
4.  **Deliverable:** Code for the LoRA layer class and training loop.

### **Part 2: Preference & Reasoning Logic (OPTIONAL / +Credit)**
*Due to hardware variance, full training is optional. Implementation verification is required for credit.*

1.  **DPO Loss (Optional +10%):**
    *   Implement the DPO Loss function manually.
    *   **Validation:** Run a **single batch** (2 chosen, 2 rejected). Print the loss value. Ensure it is non-zero and changes when you swap chosen/rejected.
    *   *Full Credit:* Complete training on 100 pairs (recommended for GPU users).
2.  **GRPO Logic (Optional +20%):**
    *   Implement the **Rule-Based Reward Function** and GRPO advantage calculation.
    *   **Validation:** Run **1 step** of the GRPO loop with Group Size $G=2$. Verify that the advantage calculation returns valid numbers and the reward function correctly identifies correct/incorrect answers.
    *   *Full Credit:* Complete training on 50 math problems (recommended for GPU users).
3.  **Distillation (Bonus +10%):**
    *   Use your tuned model as a **Teacher** to generate 50 reasoning traces.
    *   Train a fresh base model (**Student**) on this data.
    *   Compare Student inference speed (tokens/sec) and accuracy against the Teacher.

## **7. Technical Constraints & Tips**
1.  **Time Management:** If training takes > 3 hours, reduce your dataset size. We grade on *implementation correctness*, not state-of-the-art accuracy.
2.  **"From Scratch" Definition:** You are allowed to use `torch.optim` for the optimizer and `transformers` for model loading. However, the **adaptation layers (LoRA)** and **Loss Functions (DPO/GRPO)** must be written by you in PyTorch (AI-assisted is allowed but must be understood).
3.  **Memory Optimization:** Use **Gradient Checkpointing** and **Mixed Precision (FP16/BF16)** if your hardware supports it. For CPU-only, use `float32` with small batch sizes.
4.  **Verification:** For GRPO, ensure your answer parser is robust (e.g., handle "The answer is 5" and "5" equally).

## **9. Common Pitfalls to Avoid**
*   **Blind Trust:** Copy-pasting AI code without checking if base weights are frozen in LoRA.
*   **Over-scoping:** Attempting to train on full datasets leading to timeout. **Stop early** if you exceed the time limit; submit partial results with explanation.
*   **Verification Gap:** Failing to document *how* you checked the AI's code. The log must show you understood the logic.
*   **GRPO Sampling:** Setting the temperature too low during sampling. If all $G$ responses are identical, the advantage will be zero, and the model will not learn.
