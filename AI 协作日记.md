| 环境配置与 PyTorch 安装             | `import torch` 报错 `ModuleNotFoundError`，`nvidia-smi` 显示 CUDA 13.0、RTX 5090，如何安装 PyTorch？ | 先后尝试 pip 安装 cu126、conda 安装，遇到 `Bus error`；AI 建议改用 pip 安装 cu130 版本，最终成功。 | 执行 `python -c "import torch; print(torch.cuda.is_available())"` 输出 `True`，无报错。 | CUDA 驱动向后兼容性；PyTorch 不同 CUDA 版本的安装方式；容器环境特殊问题（如 `Bus error`）的排查思路。 |
| LoRA 训练脚本调试                   | 运行 `train_lora_strategies.py` 时反复出现维度不匹配、NaN 损失、`Attempting to unscale FP16 gradients` 等错误。 | AI 多次修正 `collate_fn` 的 padding 逻辑（左侧填充、labels 对齐）；加入 logits 截断、降低学习率、梯度裁剪、跳过 NaN batch；将 `torch.cuda.amp` 替换为 `torch.amp`，并强制 LoRA 参数使用 FP32。 | 全量数据训练（52k 条）稳定运行，无 NaN 报错，loss 正常下降至约 1.5。 | 动态 padding 与 labels 对齐的重要性；NaN 损失的常见来源（溢出、未对齐）及防护措施；新版 PyTorch 混合精度 API 的使用；GradScaler 对参数 dtype 的要求。 |
| DPO 训练实现与梯度修复              | `dpo_train.py` 运行无输出，或报错 `element 0 of tensors does not require grad`。 | 发现脚本缺少 `main()` 函数，提供完整 DPO 脚本；梯度错误源于策略模型 logp 被 `torch.no_grad()` 包裹，AI 分离策略模型（保留梯度）和参考模型（冻结）的计算函数，移除多余 `.detach()`。 | 训练正常开始，进度条显示 loss 逐步下降，无梯度报错。         | DPO 训练中策略模型与参考模型的梯度处理差异；`log_softmax` 计算 logp 的正确梯度保留方式；`detach()` 的正确使用场景。 |
| GRPO 评估与优化                     | “为什么 GRPO 采样全对，优势全是 0？” 希望增加问题难度、设计更细粒度奖励、调整采样温度。 | AI 给出三个独立评估脚本：改用 GSM8K 数学题（`grpo_gsm8k.py`）、设计包含答案正确+推理步骤+格式的细粒度奖励（`grpo_finereward.py`）、开放 `temperature` 和 `top_p` 参数（`grpo_temperature.py`）。并指导离线下载 GSM8K 的方法。 | 运行 GSM8K 脚本后多数回答错误，奖励混合，优势值正负分明；细粒度奖励脚本输出奖励值包含小数；温度调整后回复多样性明显变化。 | 奖励信号对 GRPO 的重要性；难度、细粒度奖励、采样温度如何影响优势计算的有效性；GSM8K 数据集的加载与答案提取。 |
| 知识蒸馏：从 NaN 到 KL 散度稳定训练 | `distill.py` 反复出现 NaN，如何解决？                        | AI 分析原因为教师 logits 极端值导致 KL 散度溢出，尝试了 logits 截断、温度调整、学习率降低、混合精度均无效后，建议先改用 SFT 完成基础任务，并给出修复后支持 KL 散度的 `distill_kd.py` 脚本（添加 logits 截断保护、NaN 检测跳过等）。 | 最终版本 `distill_kd.py` 在 200 条数据上稳定训练，Epoch 1-3 平均 Loss 分别为 0.2754, 0.0500, 0.0500，无 NaN，生成测试正确回答简单问题。 | 知识蒸馏中 KL 散度的数值稳定性极为重要；通过限幅 logits 可以避免溢出；实现时需添加 NaN 检测和跳过机制以保证训练不中断。 |
